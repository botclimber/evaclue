Reviews Service

Token based

For some requests a token is mandatory and so in the header the following parameter must exist. In this documentation token mandatory requests are mention as

Token required



Header and required keys

Header Name Description
r-access-token token goes as value of this parameter (r stands for rentify)
Keys type description
userId number User unique Id
userType string user type for authorization purposes
email string User personal email for whatever purposes

Search

Parameter

Field Type Description
city string

Some City
street string

some Street
nr string

Some building number

Success 200

Field Type Description
address dictionary

locations array[dictionaries]
